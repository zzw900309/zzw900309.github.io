使用HEXO和GITHUB搭建个人博客

写在建站前：

很久之前就想开一个关于自己的博客，来记录自己的生活，在许多年里面也试过许多的博客工具，从最基础的新浪博客、人人小站、网易 LOFTER，再到自己建站的 WordPress，却一直因为各类工具有着这样那样的问题而没有坚持下来。

经过这么多轮反反复复的尝试，我也逐渐明确了自己真正的需求：

  • 不使用第三方工具。自己建站，既是保证了灵活性，也避免了因为第三方工具出现各种问题，带来巨大的迁移成本
  • 费用尽可能低。从服务、存储、流量三个环节来降低费用
  • 建站简便，维护简单。WordPress 被放弃的原因就是因为越来越臃肿的产品功能，MVC 分离的结构，给后期的维护和错误修复造成了巨大的困难

一年前接触到 Github Page 之后一直觉得使用 Github 这个全球最知名的代码管理工具来存储网站是一个非常明智的选择，但是苦于没有合适的前端工具，来降低建站的成本，而最近，我发现 Hexo 完全满足了我使用 Github 来管理个人网站的需求，于是便马上投入建站，使用半天时间完成了基础的建站工作

在 _config.yml 中修改网站配置

修改网站的基本信息：

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述，一般用于 SEO，通常都包含网站的关键词
keyword 网站的关键词，使用半角逗号进行分隔
author 网站的作者
language 网站使用的语言，通常是 en 或 zh-CN
timezone 网站的时区，默认使用电脑的时区，中国的时区是 Asia/Shanghai

修改网址信息：

参数 描述 默认值
url 网址
root 网站根目录
permalink 文章的永久链接格式 :year/:month/:day/:title/
permalink_defaults 永久链接中各部分的默认值

如果网站是存放在子目录中，例如 http://yoursite.com/blog，那么 url 需要设为 http://yoursite.com/blog ，同时把 root 设为 /blog/

修改主题信息：
